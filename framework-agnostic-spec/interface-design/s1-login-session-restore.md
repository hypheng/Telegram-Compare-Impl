# S1 UI 设计: 登录与会话恢复

## 涉及屏幕

- `UI-S1 Launch / Session Restore`
- `UI-S2 Login`
- `UI-S3 Chat List Entry Shell`

`UI-S3` 在本切片只承担“登录或恢复成功后的主壳落点”职责，不代表 `S2 会话列表` 已完成验收。

## UI-S1 Launch / Session Restore

### 主任务

让用户在冷启动的前一秒内知道应用正在恢复会话，而不是停留在无反馈空白页。

### 信息层级

1. 产品标题或当前上下文，例如 `Telegram Compare`。
2. 恢复进度文案，例如 `正在恢复上次会话...`。
3. 次级说明，例如 `如果没有保存的会话，将进入登录`。

### 状态

| 状态 | 视觉反馈 | 行为 |
|---|---|---|
| restoring | 居中进度 + 简短说明 | 自动触发恢复检查 |
| restored | 过渡到主壳入口 | 不停留在本页 |
| failed | 顶部错误 banner 后回退登录页 | 保留重试恢复入口 |

### 低保真结构

```text
+----------------------------------+
|          Telegram Compare        |
|                                  |
|              [spinner]           |
|        正在恢复上次会话...        |
|   如果没有保存的会话，将进入登录   |
+----------------------------------+
```

## UI-S2 Login

### 主任务

让用户在最少字段下完成 mock 登录，并在输入、提交、失败时都得到即时反馈。

### 信息层级

1. 页面标题 `登录 Telegram Compare`。
2. 错误 banner 或恢复失败提示。
3. 手机号输入框。
4. 验证码输入框。
5. 主按钮 `继续`。
6. demo 提示和辅助说明。

### 状态

| 状态 | 视觉反馈 | 行为 |
|---|---|---|
| idle | 表单可编辑，主按钮可点击 | 默认态 |
| submitting | 主按钮 loading，表单暂时禁用 | 避免重复提交 |
| invalid-input | 字段下方或顶部提示 | 不清空用户已输入内容 |
| login-failed | 顶部错误 banner + 可重试 | 保持 CTA 可再次点击 |

### 低保真结构

```text
+----------------------------------+
| 登录 Telegram Compare            |
| [恢复失败提示，可选]              |
| 手机号                            |
| [+86 138 0000 0000____________]  |
| 验证码                            |
| [2046__________________________] |
| [            继续               ] |
| Demo 环境固定验证码: 2046         |
+----------------------------------+
```

### 交互要求

- 手机号与验证码都需要在点击主按钮前做同步校验。
- 恢复失败时，登录页顶部要承接错误文案，而不是吞掉启动阶段的失败原因。
- 提交成功后应直接进入主壳，不需要再弹确认框。

## UI-S3 Chat List Entry Shell

### 主任务

证明用户已经进入主壳，并能看到当前会话摘要和最小主导航承接位。

### 信息层级

1. 顶部标题 `Chats`。
2. 当前会话摘要，例如用户名称和手机号。
3. 主壳说明，例如 `已恢复会话` 或 `刚刚登录`。
4. 最小 mock chat rows。
5. 次级操作 `退出登录` 和调试入口。

### 状态

| 状态 | 视觉反馈 | 行为 |
|---|---|---|
| entry-ready | 显示当前会话与列表入口壳 | 默认态 |
| logout-pending | 次级按钮 loading 或禁用 | 清理会话后回到登录页 |

### 低保真结构

```text
+----------------------------------+
| Chats                            |
| @haifeng  +86 138 0000 0000      |
| 已恢复会话                        |
| -------------------------------- |
| Telegram Compare   Bootstrap...  |
| AI Infra            Need debug   |
| -------------------------------- |
| [退出登录] [写入失效会话用于测试]  |
+----------------------------------+
```

## 关键交互

- 启动恢复是自动触发动作，不要求用户点击。
- 恢复失败后，登录页要保留一个 `重试恢复` 次级入口，便于重复验证恢复分支。
- `退出登录` 必须清理本地会话并回到登录页。
- `写入失效会话用于测试` 仅作为 demo/debug 入口，用于下次冷启动验证失败路径。

## Accessibility

- 输入框需要稳定 label，不能只依赖 placeholder。
- loading、错误、登录成功反馈都要可被辅助功能读出。
- 主按钮和次级按钮至少满足平台推荐点击面积。
- reduced motion 下只保留淡入淡出，不做明显位移动画。

## 视觉映射

- 启动页和登录页保持 Telegram 的轻量浅色背景，不引入装饰性品牌图。
- 输入框使用圆角浅灰底，主 CTA 使用 Telegram 风格蓝色强调。
- 主壳入口保持平面白底和细分隔线，避免提前做卡片化列表。
