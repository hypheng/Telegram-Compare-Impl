# Interface Design

这里放 framework-agnostic 的界面与交互设计资产。

这些文件的目标不是替代 Figma，而是在仓库里保留一份可 diff、可 review、可关联实现的设计真相源。

## 内容

- `screen-inventory.md`: 页面和关键状态清单
- `interaction-states.md`: 关键交互和状态矩阵
- `visual-system.md`: 视觉语言、排版、色彩、间距、组件原则
- `design-delivery-workflow.md`: 从线框到 handoff 的工作流

## 规则

- 先定义信息层级和状态，再定义视觉样式
- 先定义核心路径，再扩展边角场景
- 设计交付必须能映射到 Android / iOS / CJMP / KMP 实现
