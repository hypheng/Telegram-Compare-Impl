# S8 Contacts 联系人

## 切片目标

为 Telegram Compare 增加一个真实可使用的 `Contacts` 联系人切片，让用户能够浏览联系人、搜索联系人，并从联系人直接打开已有聊天或发起一条新聊天主路径。

## 用户价值

- 用户可以像 Telegram 一样通过联系人而不是只通过既有聊天列表来发起会话。
- 用户可以快速搜索联系人姓名或手机号，减少长列表中的定位成本。
- 用户点击联系人后，若已有会话则直接进入；若还没有会话，则可以创建一条新会话并进入聊天详情。

## 本切片范围

- 把当前底部导航中无价值的 `Calls` 占位替换为真实的 `Contacts` tab。
- 页面实现 `UI-S9 Contacts`:
  - 顶部轻量导航区
  - 固定搜索框
  - 联系人列表
  - 固定底部导航
- 联系人列表至少展示:
  - avatar label
  - display name
  - phone number
  - status / availability
- 支持联系人搜索:
  - name
  - phone number
- 点击联系人时需要支持两条路径:
  - 若联系人已有关联聊天，则打开已有 `Chat Detail`
  - 若联系人还没有聊天，则创建一条新的单聊并进入 `Chat Detail`
- 新聊天创建成功后:
  - 同一联系人在联系人页内应刷新为 `已有聊天`
  - 聊天列表后续应能看到对应会话
- 从联系人打开聊天后，系统返回应回到联系人页，并保留搜索上下文。
- 覆盖 `loading`、`ready`、`empty`、`failed` 四类主要状态。
- 联系人页主壳必须撑满视口，顶部和底部导航固定，只有列表 viewport 可滚动。

## 暂不纳入本切片

- 读取系统通讯录权限、联系人授权弹窗和本机联系人导入。
- 邀请非 Telegram 用户、短信分享和系统分享面板。
- 多选联系人、群组创建、标签页筛选和联系人排序设置。
- 真实头像资源下载、联系人编辑和删除。
- 联系人在线状态的实时服务端同步。

## 交付假设

- 当前阶段允许继续使用 fixture-backed / local-first 数据。
- 联系人数据可以复用 demo 账户语义和现有聊天 fixture，但联系人列表必须独立于聊天列表暴露。
- “新聊天”允许基于 fixture 生成新的单聊与一条初始消息，不要求连接真实后端。
- 由于真实 Figma 文件仍未闭环，本轮继续允许以 repo-side handoff brief 代理 `D2`，但必须在 acceptance 和 parity 中显式记录。

## 核心流程约束

### 从主壳进入联系人

1. 用户在底部导航点击 `Contacts`，或在会话列表顶部点击 `写消息`。
2. 应用进入独立 `UI-S9 Contacts`。
3. 联系人页先显示 loading，再进入 ready、empty 或 failed。

### 搜索联系人

1. 用户在联系人搜索框输入姓名或手机号。
2. 列表在当前页内轻量更新，不跳出联系人页。
3. 若没有结果，显示联系人专属空态，而不是系统错误。

### 打开已有聊天

1. 用户点击已存在聊天映射的联系人。
2. 应用直接进入对应 `Chat Detail`。
3. 从聊天详情返回时，应回到联系人页，并保留原搜索上下文。

### 发起新聊天

1. 用户点击尚未有聊天的联系人。
2. 应用创建新的单聊 fixture，并进入 `Chat Detail`。
3. 返回联系人页时，同一联系人应刷新为 `已有聊天`。
4. 新聊天之后应能在后续聊天列表中看到对应会话。

## 领域落点

- `ContactSummary`: 联系人列表展示对象
- `ContactsQuery`: 联系人搜索关键词
- `ContactsLoadResult`: `Success`、`Empty`、`Failed`
- `OpenContactChatResult`: `Success`、`Failed`
- `ContactsRepository`: 联系人加载、搜索和打开聊天边界

## 对 KMP 的要求

- `shared-domain` 需要定义联系人模型、加载 / 打开聊天 use cases 和 repository 边界。
- `shared-data` 需要提供 fixture-backed 联系人实现，并在新聊天路径上回写到当前 demo 聊天数据。
- `androidApp` 需要新增 `Contacts` screen、底部导航跳转和返回路径。
- 联系人页必须沿用当前 Telegram 风格约束，不能退化为列表 demo 或 debug 表单页。
