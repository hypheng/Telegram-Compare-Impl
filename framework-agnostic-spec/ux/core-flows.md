# Core UX Flows

## F1 首次进入与登录

1. 启动应用
2. 判断是否已有会话
3. 无会话则进入登录
4. 登录成功后进入会话列表

成功标准:

- 状态反馈清晰
- 异常可恢复
- 不暴露平台实现差异到产品语义

补充说明:

- 如果存在有效会话，启动后应直接走恢复路径，而不是先闪一下登录页。
- 如果本地会话失效，必须把失败原因带到登录页，并允许用户直接重新登录。
- `S1` 的成功落点可以是 `Chat List Entry Shell`，用于证明主壳进入成功；完整会话列表能力仍由 `S2` 验收。

## F1.5 会话列表浏览与刷新

1. 进入 chat list
2. 浏览会话标题、预览、时间和未读
3. 通过搜索定位目标会话
4. 下拉刷新列表
5. 点击目标会话进入下一步承接

成功标准:

- 列表默认态具备 Telegram 风格层级
- 搜索和刷新不打断主壳结构
- 空态和错误态都可恢复

## F2 打开会话并发送文本

1. 从列表进入单聊
2. 加载最近消息
3. 输入文本并发送
4. 展示发送中 / 成功 / 失败状态

成功标准:

- 发送反馈及时
- 失败支持重试
- 滚动和输入区行为稳定

## F3 重启恢复

1. 关闭应用
2. 重新打开
3. 恢复会话与最近上下文

成功标准:

- 能快速恢复到上次主路径
- 不要求立刻拿到全部历史消息，但要有清晰占位
- 恢复成功时，要让用户知道当前内容来自本地缓存，可能不是最新内容

补充说明:

- 如果最近停留在 chat detail，恢复成功后应直接回到该详情，而不是先闪回 chat list。
- logout 或显式清空本地缓存后，重启只恢复 session，不恢复旧聊天上下文。

## F4 搜索

1. 发起搜索
2. 展示结果
3. 点击结果跳转到会话或消息位置

成功标准:

- 搜索结果必须区分会话命中和消息命中
- 从结果进入详情后，返回时保留搜索上下文
- 搜索页保持固定顶部和固定搜索框，不退回整页滚动

## F4.5 设置与个人资料

1. 从底部导航进入设置
2. 查看当前账号资料
3. 调整本地偏好
4. 必要时执行 logout

成功标准:

- 设置页信息层级清晰，不退化成 debug 面板
- 偏好项切换后立即可见
- 冷启动后偏好项仍然保留

## F4.7 媒体消息

1. 打开会话详情
2. 查看已有图片消息
3. 通过 media picker 发送新图片
4. 冷启动后恢复到含媒体消息的详情

成功标准:

- 媒体消息表达明显区别于纯文本
- media picker 不破坏详情页固定顶部和 composer 的边界
- 发送后的媒体消息会回写到列表与恢复路径

## F5 AI 助手增强

候选能力:

- 总结当前会话
- 生成回复草稿
- 从上下文中抽取待办

要求:

- AI 能力必须可开关
- 不影响基础聊天主流程
- 要能单独度量 AI 功能带来的开发复杂度
