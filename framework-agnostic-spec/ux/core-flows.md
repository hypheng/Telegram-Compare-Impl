# Core UX Flows

## F1 首次进入与登录

1. 启动应用
2. 判断是否已有会话
3. 无会话则进入登录
4. 登录成功后进入会话列表

成功标准:

- 状态反馈清晰
- 异常可恢复
- 不暴露平台实现差异到产品语义

补充说明:

- 如果存在有效会话，启动后应直接走恢复路径，而不是先闪一下登录页。
- 如果本地会话失效，必须把失败原因带到登录页，并允许用户直接重新登录。
- `S1` 的成功落点可以是 `Chat List Entry Shell`，用于证明主壳进入成功；完整会话列表能力仍由 `S2` 验收。

## F2 打开会话并发送文本

1. 从列表进入单聊
2. 加载最近消息
3. 输入文本并发送
4. 展示发送中 / 成功 / 失败状态

成功标准:

- 发送反馈及时
- 失败支持重试
- 滚动和输入区行为稳定

## F3 重启恢复

1. 关闭应用
2. 重新打开
3. 恢复会话与最近上下文

成功标准:

- 能快速恢复到上次主路径
- 不要求立刻拿到全部历史消息，但要有清晰占位

## F4 搜索

1. 发起搜索
2. 展示结果
3. 点击结果跳转到会话或消息位置

## F5 AI 助手增强

候选能力:

- 总结当前会话
- 生成回复草稿
- 从上下文中抽取待办

要求:

- AI 能力必须可开关
- 不影响基础聊天主流程
- 要能单独度量 AI 功能带来的开发复杂度
